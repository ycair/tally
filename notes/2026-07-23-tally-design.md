# 一筆 / Tally — 設計規格書

> 最後更新：2026-07-23

## 0. 專案概述

- **名稱**：一筆 / Tally
- **定位**：一款只屬於使用者的個人記帳 iOS app
- **平台**：iPhone（以 iPhone 17 Pro Max 為測試基準）
- **框架**：SwiftUI + SwiftData（iOS 17+）
- **資料策略**：全本機儲存，不上雲端，不使用 CloudKit
- **GitHub**：https://github.com/ycair/tally
- **工作目錄**：`/Volumes/YCAIR/Tally`
- **視覺風格**：淺色系簡約（Light, minimal）
- **核心精神**：一眼看到今天剩多少、快速記帳、財務月週期

---

## 1. 時間系統

### 1.1 財務月定義

財務月從每月 **15 日** 開始，至下月 **14 日** 結束。

```
例：7/15 ~ 8/14 為一個完整財務週期
```

### 1.2 影響規則

- 收入新增：只影響當前財務週期內「當日及之後」的天數，**不回溯過去**
- 固定花銷：依照實際發生日期，自然落入當時所在的財務週期
- 跨日結算：每次開啟 App 時檢查是否跨日，自動結算前一日的盈餘或超支

## 2. 資料模型

### 2.1 Expense（一筆消費）

```swift
enum ExpenseSource: String, Codable {
    case dailyBudget            // 從每日額度支出（預設）
    case jar(String)            // 從指定零錢罐支出（罐 ID）
}

@Model
class Expense {
    var name: String             // 項目名稱（必填）
    var amount: Decimal          // 總金額（必填）
    var date: Date               // 消費時間（必填，預設現在）
    var receiptNumber: String?   // 發票號碼（選填，未來接電子發票）
    var location: String?        // 地點（選填）
    var source: ExpenseSource    // 付款來源（預設 dailyBudget）
    var lineItems: [LineItem]    // 消費細項（選填）

    // 計算屬性
    var lineItemsTotal: Decimal  // 細項金額合計（若無細項則 = amount）
}
```

### 2.2 LineItem（消費細項）

```swift
@Model
class LineItem {
    var name: String
    var amount: Decimal
    var parent: Expense?
}
```

### 2.3 FixedCost（固定花銷）

```swift
enum FixedCostType: String, Codable {
    case monthly       // 房租、固定月付
    case yearly        // 汽車燃料稅，一年一次
    case installment   // 分期付款，有明確終止月份
    case scheduled     // 已知未來某月某日要扣
}

enum ReserveMode: String, Codable {
    case reserved      // 已預留，從本月額度扣
    case notYetReserved // 尚未預留，從該月份額度扣
    case amortized     // 均攤到數個月份
}

@Model
class FixedCost {
    var name: String                 // 項目名稱
    var amount: Decimal              // 總金額／每期金額
    var type: FixedCostType

    // 週期
    var startMonth: Int?             // installment: 第幾期開始
    var endMonth: Int?               // installment: 到第幾期結束
    var dueMonth: Int?               // yearly/scheduled: 發生月份（日曆月）
    var dueDay: Int?                 // scheduled: 指定日期

    // 均攤
    var amortizeToMonthly: Bool      // yearly 是否均攤到每月
    var reserveMode: ReserveMode?    // scheduled 的預留模式
    var amortizeMonths: [Int]?       // scheduled amortized: 攤到哪幾個財務月

    // 專款專用（必填）
    var bankCode: String             // 銀行代碼
    var accountNumber: String        // 帳號
    var hasDeposited: Bool           // 是否已存入
}
```

### 2.4 IncomeEvent（收入事件）

```swift
enum IncomeDestination: String, Codable {
    case budget                 // 均攤至本月額度（預設）
    case jar(String)            // 存入指定零錢罐（罐 ID），不影響每日額度
}

@Model
class IncomeEvent {
    var name: String                // e.g. "外送薪資"
    var amount: Decimal             // 400
    var date: Date                  // 發生日期
    var destination: IncomeDestination  // 預設 .budget
}
```

只有 `destination = .budget` 的收入才進入每日額度均攤；`.jar` 的收入直接存入指定零錢罐，不影響額度計算。

### 2.5 MoneyJar（零錢罐）

```swift
@Model
class MoneyJar {
    var name: String             // "旅遊基金"
    var bankCode: String         // 銀行代碼（必填）
    var accountNumber: String    // 帳號（必填）
    var balance: Decimal         // 系統追蹤餘額
    var createdAt: Date          // 建立時間
}
```

零錢罐 vs 存款（AppSettings.savingsBalance）：
- **存款**：每日剩餘額度自動累積，不關聯特定銀行帳戶
- **零錢罐**：使用者手動建立，有指定銀行帳戶，只能由 `.jar` 收入事件存入、或由 `source = .jar` 的消費支出

### 2.6 DailyBudget（每日額度記錄）

```swift
@Model
class DailyBudget {
    var date: Date              // 日期
    var baseAmount: Decimal     // 當日基礎額度（收入均攤 ─ 固定花銷均攤）
    var penalty: Decimal        // 前期超支懲罰扣款
    var spent: Decimal          // 當日已消費總額
    var leftover: Decimal       // 結算後剩餘（正=存、負=超支）
    var status: DayStatus       // green / white / red
}

enum DayStatus: String, Codable {
    case green   // 有剩餘 → 餘額轉入存款
    case white   // 剛好打平
    case red     // 超支 → 懲罰均攤至後續天數
}
```

### 2.7 全域設定

```swift
@Model
class AppSettings {
    // 單例模式
    static var shared: AppSettings

    var savingsBalance: Decimal        // 系統追蹤的存款餘額
    var verifiedBalance: Decimal?      // 使用者對賬確認的實際餘額
    var lastVerifiedDate: Date?        // 最後對賬日期
}
```

## 3. 每日額度計算邏輯

```
對於財務週期內每一天 D：

1. 基礎額度 = (本週期內已生效的 IncomeEvent 均攤至 D 的部份)
             ─ (本週期內固定花銷均攤至 D 的部份)

2. 今日可用額度 = 基礎額度 ─ 前期超支懲罰（若有）

3. 每日結束結算：
   ├── leftover > 0  → status = green  → 餘額自動存入「存款帳戶」
   ├── leftover = 0  → status = white  → 無事發生
   └── leftover < 0  → status = red    → ｜leftover｜÷ 本週期剩餘天數，
                                          懲罰均攤至後續每日額度
```

### 3.1 收入事件計算範例

```
週期：7/15 ~ 8/14（共 31 天）
7/31 新增收入 +400 元（外送薪資）

  影響範圍：7/31 ~ 8/14（共 15 天）
  每日均攤：400 ÷ 15 ≈ 26.67（取整至元，餘數放第一天）

  7/31 基礎額度 +27
  8/1 ~ 8/14 每日基礎額度 +26
```

### 3.2 固定花銷均攤範例

```
燃料稅 7,440 元（每年 4 月），均攤 12 個財務月

  每個財務月均攤：7,440 ÷ 12 = 620
  落入的財務月包括：3/15~4/14、4/15~5/14、... 共 12 個月
```

## 5. 全螢幕記帳模式

上拉後整頁覆蓋：

```
┌─────────────────────────────┐
│  取消               儲存     │
│─────────────────────────────│
│                             │
│        午餐                 │  ← 項目名稱（大字）
│                             │
│─────────────────────────────│
│                             │
│      NT$ 180                │  ← 金額（大字，自動彈數字鍵盤）
│                             │
│─────────────────────────────│
│  來源     每日額度  >        │  ← 預設每日額度，點擊可選零錢罐
│  時間     7/23 12:30  >     │  ← 預設現在
│  地點     添加快取地點  >    │  ← 選填
│  發票     輸入發票號碼  >    │  ← 選填，未來接電子發票
│─────────────────────────────│
│  ＋ 新增消費細項             │  ← 初始收起
│    ┌──────────┬──────────┐  │
│    │ 主餐     │  NT$150  │  │
│    │ 飲料     │  NT$30   │  │
│    └──────────┴──────────┘  │
│─────────────────────────────│
│                             │
│            刪除此筆         │  ← 編輯既有記錄時顯示
└─────────────────────────────┘
```

- 新增「來源」欄位：預設「每日額度」，點擊可切換至任一零錢罐
- 選擇罐子支出時不消耗每日額度，直接從該罐扣除

### 互動細節
- 打開時鍵盤自動彈出，游標在金額欄
- 項目名稱使用 `.font(.largeTitle)`
- 細項初始收起，點擊 `＋新增消費細項` 展開
- 細項可無限新增，每行可左右滑動刪除
- 儲存後 dismiss，回到首頁，金額即時更新
- 取消時若有輸入內容 → 彈出 alert 確認是否捨棄

## 6. 新增固定花銷表單

依類型不同展開不同欄位：

| 類型 | 顯示欄位 |
|---|---|
| 每月 | 名稱、金額、專款帳戶（選擇既有或新增）、是否已存入 |
| 每年 | 名稱、金額、日曆發生月份、是否均攤至 12 財務月、專款帳戶 |
| 分期 | 名稱、每期金額、第 N 期/共 M 期、起始月份、專款帳戶 |
| 預定扣款 | 名稱、金額、日期（月+日）、預留模式（已留/未留/均攤）、攤分月份（均攤時顯示）、專款帳戶 |
